# PROMPT: Fix Component Data Priority Issue

## đźŽŻ Objective

This prompt explains how to fix data priority issues in components where components display incorrect data on the client-side because they're not correctly prioritizing `tenantComponentData` (from API) over `storeData` (from Zustand store).

**Use this prompt with a component file to fix data priority and ensure correct data display.**

---

## đź“‹ Problem Statement

### Current Behavior (Incorrect):
- Components display wrong data on client-side
- All instances of the same component type show the same incorrect data
- `tenantComponentData` from API exists but is not being used
- `storeData` is being used even when `tenantComponentData` exists

### Desired Behavior (Correct):
- Each component instance displays its correct data from API
- `tenantComponentData` is prioritized over `storeData` when it exists
- Components correctly identify themselves using `id` prop

---

## đź”Ť Root Cause Analysis

### Why Components Display Wrong Data:

1. **TenantPageWrapper doesn't pass `id` prop:**
   ```typescript
   // âťŚ BAD: Missing id prop
   <Cmp {...(comp.data as any)} useStore variant={comp.id} />
   ```

2. **Component uses wrong identifier:**
   - `props.id` is `undefined` because it's not passed
   - Component falls back to `variantId` which may be incorrect
   - `getTenantComponentData()` can't find the correct component data

3. **Merge logic doesn't prioritize tenantComponentData:**
   ```typescript
   // âťŚ BAD: Uses storeData even when tenantComponentData exists
   ...(hasTenantData && isStoreDataDefault
     ? {}
     : currentStoreData), // Still uses storeData if not default
   ```

**If order is wrong, fix it:**

### Step 3: Fix Merge Logic

**Find the `mergedData` object in the component:**

```typescript
// âťŚ BAD: Current logic (doesn't prioritize tenantComponentData)
const mergedData = {
  ...defaultData,
  ...props,
  ...(hasTenantData ? tenantComponentData : {}),
  ...(hasTenantData && isStoreDataDefault
    ? {}
    : currentStoreData), // Still uses storeData if not default
};
```

**Replace with improved logic:**

```typescript
// âś… GOOD: Improved logic (prioritizes tenantComponentData)
const mergedData = {
  ...defaultData, // 1. Defaults (lowest priority)
  ...props, // 2. Props from parent component
  // If tenantComponentData exists, use it (it's from Database)
  ...(hasTenantData ? tenantComponentData : {}), // 3. Backend data (tenant data)
  // Skip store data if tenant data exists (tenant data is source of truth)
  // Only use store data if no tenant data exists
  ...(hasTenantData
    ? {} // Skip store data if tenant data exists
    : {
        ...storeData, // 4a. Store data from {componentType}States (saved data)
        ...currentStoreData, // 4b. Current store data from getComponentData (saved data)
      }),
};
```

**Key Changes:**
1. Remove `isStoreDataDefault` check - always skip store data if tenant data exists
2. Add `storeData` to mergedData when no tenant data exists
3. Ensure both `storeData` and `currentStoreData` are included when no tenant data

### Step 4: Verify getTenantComponentData Function

**Check that `getTenantComponentData()` searches by `props.id`:**

**Ensure it checks:**
- `componentId === props.id` (most reliable)
- `(component as any).id === props.id`
- `(component as any).id === uniqueId` (fallback)

## đź“š Key Concepts

### Data Priority Order:

1. **defaultData**: Lowest priority (fallback values)
2. **props**: From parent component
3. **tenantComponentData**: From API (Database) - **Highest priority when exists**
4. **storeData**: From Zustand store (only when no tenant data)

### Component Identification:

- **`variant`**: Component type/name (e.g., "halfTextHalfImage6")
- **`id`**: Unique instance identifier (e.g., "1", UUID)
- **`uniqueId`**: `props.id || variantId` (fallback to variant if no id)

### When to Use Each Data Source:

- **Use `tenantComponentData`**: Always when it exists (from API)
- **Use `storeData`**: Only when no tenant data exists
- **Never mix**: Don't use store data if tenant data exists
